#### 統一規則（footer convention SSOT）— EDITORIAL §十 canonical

5 條鐵律：

1. 每檔最多一個 `_current: vX.Y | YYYY-MM-DD_` marker
2. 不在 body heading 嵌入 version annotation（`### v1.1：規則名` → `### 規則名（v1.1）` 把 version 移到 suffix）
3. 完整 changelog → git log（文件不重複）
4. 觸發事件 narrative → reports/ 或 diary/，文件留 1 行 pointer
5. 跨檔重複教訓 → canonical pointer（李洋 #28/#29 → DNA #23）

3 個 audit 工具（grep 一行 detect）：

```bash
grep -c "^_\(current\|版本\)" file.md           # 應 ≤ 1
grep -nE "^#{1,4} v[0-9]+\.[0-9]+\b" file.md   # 應 0
grep -nE "誕生事件|觸發背景" file.md             # 應 0 或 condensed pointer
```

何時可破例：

- 新生 pipeline（< 30 天）：誕生事件 inline H2 段保留 motivation hook 合理（30 天後重 audit）
- 意義保留檢查：「為什麼這條規則存在」對讀者理解規則必要 → 留；只是「何時誕生」歷史 → 搬
