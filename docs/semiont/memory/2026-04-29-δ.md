### 學到什麼

教訓萃取進 LESSONS-INBOX (詳見 §神經迴路候選下方)。最重要：

1. **SolidJS IIFE inside `<Show>` is reactivity poison** — `<Show when={X}>{(() => { const c = X; return JSX })()}</Show>` 看似 idiomatic 但 IIFE 是 setup-time once，capture 後 frozen。直接 inline reactive accessor 才是慣例。React/Vue 也類似 pattern。**通用反射候選**。

2. **Per-engine model lookup table 勝過 if/else compute path** — cheyu 昨晚的 nested table 設計乾淨、data-driven、易擴充新 engine 不用改 control flow。我 instinctive 的 if/else 也 work 但每加一個 engine 要動 3 處 (lookup, switch, ENGINE_ELIGIBLE_TIER 重複定義 model defaults)。**Data > Logic** when mapping is the point.

3. **codex CLI metadata.model field is empty** — 跟 claude CLI 相反，codex stream JSON 不在 message 內 expose effective model。要在 UI 顯示 model 必須從 task.inputs 推或 fallback "auto" label。**Cloud agent 應該強制 expose effective model id 在 metadata header** — SOP 候選。

4. **codex ratio 2.53 is structural, not random** — Sonnet 1.47 / codex 2.53 / 同篇任務同 prompt。codex 對 zh prose 有 elaborate 結構傾向。**faithfulness-over-polish 需要硬 gate (ratio > 1.6 = REJECT)，不是 warn**。或 prompt level 加 length cap explicit instruction。

5. **觀察者連發 P0 = scaffolding, not interruption** — 第 N 次驗證 (DNA #15 + 2026-04-19 β LESSONS-INBOX 已存)。本 session 7 個 wave 全部「邊做邊加」，每個 callout 都是「該想到的下一件事」。
